# IAM Hardening Checklist

- [ ] MFA enabled for all human users
- [ ] Root account access keys disabled
- [ ] No long-term admin credentials for routine operations
- [ ] Least-privilege policies applied (no unnecessary wildcards)
- [ ] Role trust policies limited to approved principals
- [ ] Access keys rotated and inactive keys removed
- [ ] Cloud audit logging enabled and retained
- [ ] Alerts configured for high-risk IAM actions
- [ ] Quarterly access certification documented
